🦆 docling-serve

HTTP conversion API over docling.rs — convert documents to Markdown, docling JSON, DocLang archives, or chunk records.

Try it

PDF/image inputs run the ML pipeline — the first conversion loads the models (tens of seconds), repeats are fast. Markdown responses stream in as they convert.

  





API



  
MethodPathDescription

  
POST/v1/convert
      Convert an upload (multipart/form-data, a file part — the filename's
      extension selects the input format) or a URL (application/json:
      {"url": …, "file_name"?: …}).

  
GET/healthLiveness probe.

  
GET/readyReadiness probe — 503 until --warmup finishes loading the ML models.





Options


As query parameters, multipart text fields, or JSON keys (body wins over query):



  
OptionValuesDescription

  
tomd (default) | json | dclx | chunks
      Markdown (streamed) · docling DoclingDocument JSON · DocLang OPC archive (download) · chunker records (hierarchical + hybrid when a tokenizer is installed).

  
stricttrue/falseCleaner strict Markdown dialect instead of byte-for-byte docling-legacy output.

  
imagesplaceholder (default) | embeddedMarkdown picture handling (embedded = base64 data URIs).

  
no_ocrtrue/falsePDF/image: skip layout/OCR/TableFormer entirely; emit the embedded text layer only.

  
no_table_formertrue/falsePDF/image: geometric table reconstruction instead of the TableFormer model.

  
fetch_imagestrue/falseHTML/EPUB: resolve external <img src> and embed the bytes.





Examples


curl -F file=@paper.pdf  localhost:5001/v1/convert                 # Markdown (streams)
curl -F file=@report.docx 'localhost:5001/v1/convert?to=json'      # docling JSON
curl -F file=@sheet.xlsx  'localhost:5001/v1/convert?to=dclx' -OJ  # DocLang archive
curl -F file=@page.html   'localhost:5001/v1/convert?to=chunks'    # chunk records
curl -H 'content-type: application/json' \
     -d '{"url": "https://example.com/doc.pdf", "to": "md"}' \
     localhost:5001/v1/convert                                     # convert a URL
